# ColorScripts-Enhanced Module Help

## about_ColorScripts-Enhanced

### SHORT DESCRIPTION
Улучшенные PowerShell ColorScripts с высокопроизводительной системой кэширования для отображения красивого ANSI искусства в вашем терминале.

### LONG DESCRIPTION
ColorScripts-Enhanced - это модуль PowerShell, который предоставляет коллекцию из 450++ красивых ANSI colorscripts для отображения в терминале. Он включает интеллектуальную систему кэширования, которая обеспечивает улучшение производительности в 6-19 раз по сравнению с традиционным выполнением скриптов.

Модуль автоматически кэширует вывод скриптов в централизованное место в вашей папке AppData, обеспечивая мгновенное отображение при последующих запусках. Файлы кэша автоматически проверяются и регенерируются при изменении исходных скриптов.

### FEATURES
- 450++ красивых готовых colorscripts
- Высокопроизводительная система кэширования (6-19 раз быстрее)
- Централизованное расположение кэша (работает из любого каталога)
- Автоматическая инвалидация кэша при изменении скриптов
- Поддержка кодировки UTF-8 для идеального рендеринга
- Случайный выбор colorscript
- Легкое обнаружение и перечисление скриптов
- Помощник профиля для однострочной интеграции запуска
- Постоянные помощники конфигурации для поведения кэша и запуска
- Рабочие процессы экспорта метаданных и scaffolding для пользовательских скриптов

### CACHE SYSTEM

Система кэширования работает автоматически:

1. **Первый запуск**: Скрипт выполняется нормально и вывод кэшируется
2. **Последующие запуски**: Кэшированный вывод отображается мгновенно (6-19 раз быстрее)
3. **Авто-обновление**: Кэш регенерируется при изменении скрипта

**Расположение кэша**: `$env:APPDATA\ColorScripts-Enhanced\cache`

**Преимущества кэша**:
- Мгновенное отображение сложных colorscripts
- Сниженное использование CPU
- Стабильная производительность
- Работает из любого каталога

### NERD FONT GLYPHS

Некоторые скрипты отображают иконки Nerd Font (глифы разработчика, разделители powerline, галочки). Установите патченный шрифт, чтобы эти символы отображались правильно:

1. Скачайте шрифт с https://www.nerdfonts.com/ (популярные варианты: Cascadia Code, JetBrainsMono, FiraCode).
2. Windows: извлеките `.zip`, выберите файлы `.ttf`, правой кнопкой → **Установить для всех пользователей**.
   macOS: `brew install --cask font-caskaydia-cove-nerd-font` или добавьте через Font Book.
   Linux: скопируйте файлы `.ttf` в `~/.local/share/fonts` (или `/usr/local/share/fonts`) и выполните `fc-cache -fv`.
3. Установите профиль терминала для использования установленного Nerd Font.
4. Проверьте глифы с помощью:

```powershell
Show-ColorScript -Name nerd-font-test
```

### TROUBLESHOOTING

#### Scripts not displaying correctly
Убедитесь, что ваш терминал поддерживает UTF-8 и ANSI escape codes:
```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
```

#### Cache not working
Очистите и перестройте кэш:
```powershell
Clear-ColorScriptCache -All
New-ColorScriptCache
```

#### Performance issues
Первый запуск каждого скрипта будет медленнее, так как он строит кэш. Предварительно постройте все кэши:
```powershell
New-ColorScriptCache
```

#### Module Not Found Error
```powershell
# Check if module is in PSModulePath
Get-Module ColorScripts-Enhanced -ListAvailable

# List available module paths
$env:PSModulePath -split ';'
```

#### Cache Corruption
Очистите и перестройте полностью:
```powershell
# Remove module from session
Remove-Module ColorScripts-Enhanced -Force

# Clear all cache files
Clear-ColorScriptCache -All -Confirm:$false

# Reimport and rebuild cache
Import-Module ColorScripts-Enhanced
New-ColorScriptCache -Force
```

#### Performance Degradation
Если производительность ухудшается со временем:
```powershell
# Check cache directory size
$cacheDir = (Get-ColorScriptConfiguration).Cache.Path
$size = (Get-ChildItem $cacheDir -Filter "*.cache" |
    Measure-Object -Sum Length).Sum
Write-Host "Cache size: $([math]::Round($size / 1MB, 2)) MB"

# Clear old cache and rebuild
Clear-ColorScriptCache -All
New-ColorScriptCache
```

#### Topic 1: Cache Strategy Selection
Различные стратегии кэширования для разных сценариев:

**Full Cache Strategy** (Optimal for Workstations)
```powershell
New-ColorScriptCache              # Cache all 450++ scripts
# Pros: Maximum performance, instant display
# Cons: Uses 2-5MB disk space
```

**Selective Cache Strategy** (Optimal for Portable/CI)
```powershell
Get-ColorScriptList -Tag Recommended -AsObject |
    ForEach-Object { New-ColorScriptCache -Name $_.Name }
# Pros: Balanced performance and storage
# Cons: Requires more setup
```

**No Cache Strategy** (Optimal for Development)
```powershell
Show-ColorScript -NoCache
# Pros: See script changes immediately
# Cons: Slower display, more resource usage
```

#### Topic 5: Scripting and Automation

```powershell
# Create reusable function for daily greeting
function Show-DailyColorScript {
    $seed = (Get-Date).DayOfYear
    Get-Random -SetSeed $seed
    Show-ColorScript -Random -Category @("Geometric", "Nature") -Random
}

# Use in profile
Show-DailyColorScript

# Create script rotation function
function Invoke-ColorScriptSlideshow {
    param(
        [int]$Interval = 3,
        [string[]]$Category,
        [int]$Count
    )

    $scripts = if ($Category) {
        Get-ColorScriptList -Category $Category -AsObject
    } else {
        Get-ColorScriptList -AsObject
    }

    $scripts | Select-Object -First $Count | ForEach-Object {
        Show-ColorScript -Name $_.Name
        Start-Sleep -Seconds $Interval
    }
}

# Usage
Invoke-ColorScriptSlideshow -Interval 2 -Category Geometric -Count 5
```

```powershell
Remove-Module ColorScripts-Enhanced
Uninstall-Module ColorScripts-Enhanced
Install-Module -Name ColorScripts-Enhanced -Force
```

#### Issue 3: Cache Not Being Used
**Symptoms**: Scripts run slowly every time
**Solutions**:
```powershell
# Verify cache exists
$cacheDir = (Get-ColorScriptConfiguration).Cache.Path
Get-ChildItem $cacheDir -Filter "*.cache" | Measure-Object

# Rebuild cache
Remove-Item "$cacheDir\*" -Confirm:$false
New-ColorScriptCache -Force

# Check for cache path issues
Get-ColorScriptConfiguration | Select-Object -ExpandProperty Cache
```

#### Issue 4: Profile Not Running
**Symptoms**: Colorscript doesn't show on PowerShell startup
**Solutions**:
```powershell
# Verify profile exists
Test-Path $PROFILE

# Check profile content
Get-Content $PROFILE | Select-String "ColorScripts"

# Repair profile
Add-ColorScriptProfile -Force

# Test profile manually
. $PROFILE
```

### FAQ

**Q: How many colorscripts are available?**
A: 450++ встроенных скриптов в нескольких категориях и с тегами

**Q: How much disk space does caching use?**
A: Примерно 2-5MB всего для всех скриптов, около 2-50KB на скрипт

**Q: Can I use colorscripts in scripts/automation?**
A: Да, используйте `-ReturnText` для захвата вывода или `-PassThru` для метаданных

**Q: How do I create custom colorscripts?**
A: Используйте `New-ColorScript` для создания шаблона, затем добавьте ваше ANSI искусство

**Q: What if I don't want colors on startup?**
A: Используйте `Add-ColorScriptProfile -SkipStartupScript` для импорта без авто-отображения

**Q: Can I use this on macOS/Linux?**
A: Да, с PowerShell 7+, который работает кросс-платформенно

**Q: How do I share colorscripts with colleagues?**
A: Экспортируйте метаданные с `Export-ColorScriptMetadata` или поделитесь файлами скриптов

**Q: Is caching always enabled?**
A: Нет, используйте `-NoCache` для отключения кэширования при разработке/тестировании

### BEST PRACTICES

1. **Install from PowerShell Gallery**: Используйте `Install-Module` для автоматических обновлений
2. **Add to Profile**: Используйте `Add-ColorScriptProfile` для автоматической интеграции запуска
3. **Pre-build Cache**: Запустите `New-ColorScriptCache` после установки для оптимальной производительности
4. **Use Meaningful Naming**: При создании пользовательских скриптов используйте описательные имена
5. **Keep Metadata Updated**: Обновляйте ScriptMetadata.psd1 при добавлении скриптов

7. **Monitor Cache Size**: Периодически проверяйте размер каталога кэша и очищайте при необходимости
8. **Use Categories/Tags**: Используйте фильтрацию для более быстрого обнаружения скриптов
9. **Document Custom Scripts**: Добавляйте описания и теги к пользовательским colorscripts
